# LED Mesh 现场测量方案对标(全站仪 / X6+DST360-X / BLK3D / 分支二相机)

> 场景:在大尺寸 LED 屏(示例 8m×3m、12m 半径弧形;规划可达 60m 弧墙)上测控制点,
> 建立统一世界坐标系,喂给 `lmt_core` 重建管线。项目精度账本按 **2mm isotropic** 记。
> 本表用于选型评估;数据来源与推断标注见文末。

## 一句话结论

- **全站仪**:精度与统一坐标系的金标准,大屏/弧墙/xR 唯一稳妥项,代价是贵、重、要专业操作。
- **X6 + DST360-X**:架构和全站仪同源,**修好了 BLK3D 的「无全局坐标系」死穴**,但横向精度差 2.5×,且**搬站即丢坐标系**——只适合 VP 级、单站测得完的小/平屏。
- **BLK3D**:工具选型错配。短距单房间立体测量的盒子,套不进「大屏 + 统一坐标系 + 2mm」,**在分支二已走的路上还更差**(基线小、量程短、深度方向最弱)。
- **分支二相机**:结构上最接近「替代全站仪」——多视角 BA 天生统一坐标系、无搬站问题、用屏自显 pattern、数据原生进管线。**卡点是真实 LED 屏实测精度尚未验证**(PoC 未做)。

## 脚注

**① 全站仪精度数字的出处。** Trimble **SX12 用户指南**(`SX12_RevA_UserGde`,2023.04)
**通篇没有精度规格数字**,只定性讲技术;精度数字在单独的 **SX12 Datasheet**(PN 022536-507B,
01/23)里。已读 datasheet,完整规格如下:

| 项 | 值 |
|---|---|
| 测角精度 | **1″(0.3 mgon)**,绝对编码器对径读数;显示分辨率 0.1″ |
| 双轴补偿器 | 精度 0.5″(0.15 mgon),范围 ±5.4′ |
| 测距 prism Std | **1mm + 1.5ppm**(1.6s);Tracking 2mm+1.5ppm |
| 测距 DR Std | **2mm + 1.5ppm**(1.2s) |
| 量程 | prism 5,500m;DR(白卡)800m / (灰卡)450m |
| Autolock 测角精度 | 1″;追踪范围 traverse 800m / 360° 棱镜 300~700m |
| 扫描 | 角精度 5″(1.5 mgon);**3D 点精度 @100m 2.5mm**;测程噪声 @50/120/200m 1.5mm、@300m 2.5mm;26.6kHz |
| EDM | 脉冲 1550nm,Class 1M;DR 发散 0.2mrad;@100m 光斑 14mm |
| 激光指示器 | 绿光 520nm,可调焦,Class 1;@50m 光斑 3mm |
| 系统 | IP55;−20~50℃;7.5kg;横轴高 196mm;前镜口径 56mm |

**对本场景的意义:** 测角 1″ ≈ 0.0003°,比 DISTO/DST360-X 的 ±0.1° 精约 **360×**;在 LED 墙
5~10m 距离下,横向角度贡献仅亚毫米级,3D 点精度由 EDM 主导 → **~1mm**(厂商自报 100m 才
2.5mm)。项目记账 **2mm** 是保守值。

**② BLK3D Desktop 软件不改变结论。** `BLK3D Desktop v1.2.0`(2019)是面向 AEC 文档的
处理 + CAD 导出配套:ORGANISER 管理、MEASURE 大屏量测 + Recalculate(载入 multi-shot
提精度)、3D MODEL(选配)在**单张** 3D image 上画线建模并导 DXF/DWG 3D、BIM 360 /
Publisher 云端分享。**关键缺失依旧:无多图配准 / 合并 / 控制点 / георef / BA**——3D MODEL 是
单图局部系,导出的 DXF/DWG 各自一个原点,喂不进 `adapter-total-station` 的世界系 CSV。
桌面处理也加不了基线 / 量程,深度物理硬伤不变。仅让 BLK3D 在"单张近距小/平屏"那个 niche
更好用。
