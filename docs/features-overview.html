Pipeline

给现实里的 LED 大屏拍方位、量坐标，反算出三维 mesh，导成 Disguise / Unreal / 中性坐标系能用的 OBJ。按 ① → ⑦ 顺序走；先把三个变量设好，后面每条命令复制就能直接用。

先把这几个变量设好 · 在仓库根目录下执行

LMT=./target/debug/lmt # lmt 在 PATH 上就直接写 lmt PROJECT=~/projects/my-led-wall # 换成你的项目目录 SCREEN=main # 换成你的 screen id
①

管理项目工作区

Workspace & recent projects 双线全通

每块 LED 大屏对应磁盘上一个项目文件夹，里面放着 project.yaml、测量数据、重建产物。工具会记住最近打开过哪些项目，不用每次都去翻路径。

  • 看最近项目列表：项目名、文件夹路径、上次打开时间
  • 从内置示例新建项目：curved-flat（8×4 平面）或 curved-arc（16×6 弧面）
  • 打开已有项目文件夹，自动读取 project.yaml
  • 从最近列表里去掉某个项目（不删本地文件，只是不再记录它）

CLI · 复制即用

查看最近打开过的项目

$LMT project list-recent

加进最近列表（write_safe，不需要 --yes）

$LMT project add-recent "$PROJECT" "我的弧形墙"

打开项目，读取 project.yaml

$LMT project load "$PROJECT"

从最近列表移除（destructive，本地文件不会被删）

$LMT project remove-recent <id> --yes
②

设计屏体形状

Screen geometry design 双线全通

在工具里描述这块屏长什么样：多少个 cabinet 拼在一起、每块多大、是平的还是弯的、哪些位置是空的（异形屏），还有用哪三个点来定坐标系。

  • 配置 cabinet 阵列：几列几行、每块物理尺寸（mm）、每块的像素数
  • 编辑遮罩：把不存在的 cabinet 标出来，用来做异形屏
  • 指定形状先验：flat 平面 / curved 弧面（填半径）/ 自定义
  • 标定 3 个参考点：原点、X 轴方向点、XY 平面定向点
  • 改动支持撤销 / 重做，保存时原子写回 project.yaml

CLI · 复制即用

改之前先读出当前配置

$LMT project load "$PROJECT"

预演写回过程、不实际落盘（先确认没问题再动）

$LMT project save "$PROJECT" --input project.yaml --dry-run

原子写回 project.yaml（destructive）

$LMT project save "$PROJECT" --input project.yaml --yes
③

生成施工指令卡

Field instruction card (HTML / PDF) HTML 全通,PDF 仅 GUI

给现场测量员打一张 A4 纸：画出屏体方位、标出 3 个参考点的位置、列出所有要测的目标点编号。不是给设计师看的，是给工人手里拿着用的——排版得在工地上扫一眼就能看懂。

  • 从当前项目 + 屏幕配置直接生成可打印的指引卡 HTML
  • GUI 里还能把 HTML 渲染成 PDF（macOS WKWebView / Windows WebView2）
  • CLI 拿到 HTML 之后，自己接 headless Chrome / wkhtmltopdf 渲染

CLI · 复制即用

生成可打印 A4 指引 HTML（read_only）

$LMT total-station instruction-card "$PROJECT" "$SCREEN" > card.html

headless 把 HTML 渲成 PDF（PDF 仅限 GUI）

chrome --headless --print-to-pdf=card.pdf card.html
④

导入现场测量数据

Total-station CSV → measured.yaml 双线全通

现场工程师用全站仪一个点一个点打出来的坐标，导一份 CSV 进来就能用。工具自动把 CSV 行号映射到屏上的点编号，然后产出导入报告：哪些点是实测的、哪些是补出来的、哪些是离群点、哪些缺失。

  • 把全站仪导出的 CSV 一键导入
  • 自动生成 measurements/measured.yaml + import_report.json
  • 报告分四类：实测 / 补出 / 离群 / 缺失，每类点数一目了然
  • 也支持直接加载手写的 measured.yaml

CLI · 复制即用

导入全站仪 CSV（destructive，旧文件备份为 .bak，失败自动回滚）

$LMT total-station import "$PROJECT" "$SCREEN" survey.csv --yes

复核结果：实测 / 补出 / 离群 / 缺失（read_only）

$LMT measurements load "$PROJECT/measurements/measured.yaml"
⑤

几何重建

Surface reconstruction from measured points 双线全通

这是工具的核心功能。输入测量点 + 形状先验，反算出整块屏的三维 mesh（顶点 + 三角面），同时给出质量指标：RMS 残差、最大误差、顶点数。每跑一次自动落盘，数据库里记一条 run。

  • 跑算法：输入 measured.yaml + 屏体先验，产出 mesh
  • 报告写到 reports/<时间戳>.json，完整、可重现
  • 数据库记一条 run（关联屏幕、方法、参数、质量数据）
  • 同一块屏可以跑多次，每次都是独立的 run，可以互相对比

CLI · 复制即用

先预演，确认输入路径没问题

$LMT reconstruct surface "$PROJECT" "$SCREEN" measurements/measured.yaml --dry-run

正式跑（destructive，写 report.json + 数据库一行记录）

$LMT reconstruct surface "$PROJECT" "$SCREEN" measurements/measured.yaml --yes
⑥

查看运行历史

Runs & reports 双线全通

每次重建都有记录。这里可以看所有 run，知道这块屏调了几版、每版 RMS 是多少、最后导出的是哪一版。点进去能看完整 JSON 报告（mesh 结构、质量分布、参数全在里面）。

  • 列出当前项目的所有 run：时间、屏幕、方法、RMS（mm）、顶点数、导出目标
  • 点开某行查看完整 JSON 报告
  • raw 和 canonical 两套路径 key 都能查（老项目换过路径也能找到旧 run）

CLI · 复制即用

列出 run、对比 RMS、拿 run_id（read_only）

$LMT reconstruct list-runs "$PROJECT" --screen-id "$SCREEN"

查看某条 run 的完整 JSON 报告（read_only）

$LMT reconstruct get-run-report <run_id>
⑦

导出可用模型

OBJ export · three coordinate systems 双线全通

重建完还不算结束——还得把 mesh 交给下游软件。下游有 3 种，坐标系互不兼容，所以工具一次产出三种 OBJ，用哪个取决于你对接的是哪套系统。

  • Disguise — 现场 VFX 演出系统（d3 / disguise），Y-up 右手系
  • Unreal — 虚幻引擎，Z-up 左手系，单位 cm
  • Neutral — 中性坐标系，工具默认的几何空间，用于调试 / 二次加工

CLI · 复制即用

Disguise · VFX 演出 · Y-up 右手系

$LMT export obj <run_id> disguise --yes

Unreal · 虚幻引擎 · Z-up 左手系 · cm

$LMT export obj <run_id> unreal --yes

Neutral · 中性坐标系 · 调试 / 二次加工

$LMT export obj <run_id> neutral --yes
